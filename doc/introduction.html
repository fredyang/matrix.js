Introducing Matrix.js

What is Matrix.js

Marix.js is an extensible JavaScript library that can load client side modules on demand from server. It helps you to build Single Page Application with faster loading speed, less memory consumption, and better module reusability.

What is the dependency of Matrix.js

jQuery 1.5+. It can dynamically load jQuery from CDN, if jQuery is not loaded.

Background

Today, we are trying to build large scale Single Page Application with experience of native application. This kind of application usually depends lots of modules. These modules can be are system modules like jQuery, or MV-ish library/Framework, or they can be application modules such customs widget, template, css module, localization resource, lookup data. As the application grows, the number of moduels increase. These becomes quite a challenge us. Here is a couple of problems that I have want to solve.

  1. Slow application start-up time.

    We tend to load all the modules that an application need in the entire life time of the application.

  2. Application can load module on demand it can not release module when it does not need it anymore.
  3. Application can load JavaScript files on demand only.

What is a module?

In the sense of matrix.js, it is a software construct that can be deployed to the browser such as a JavaScript file, CSS style sheet, image, localization resource, a coffee script file, even a string, a function, JSON object, or a CSS rule. A module is not necessarily mapped to downloadable unit such as a file, although in a lot of case it is.
